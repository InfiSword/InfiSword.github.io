안녕하세요. 이민혁입니다. 본 글에서는 컴퓨터 그래픽스 기술의 중추이자 현대 가속 엔진의 핵심인 GPU 렌더링 파이프라인(Rendering Pipeline)의 전체 연산 흐름을 정리하고, 소프트웨어 단에서 시도할 수 있는 데이터 구조 최적화 전략을 공유합니다.



1\. 하드웨어 렌더링 파이프라인 개요



우리가 작성한 3D 데이터(Vertex, Index 등)가 모니터 상의 2D 픽셀로 변환되기 위해 거치는 물리 가속 파이프라인은 크게 다음과 같이 구성됩니다.



\[Application] -> (Vertex Processing) -> \[Primitive Assembly] -> \[Rasterization] -> (Pixel/Fragment Shader) -> \[Merge/Output]





(괄호 ()는 셰이더 프로그래밍이 가능한 스테이지이며, 대괄호 \[]는 고정 함수 하드웨어 가속 영역입니다.)



2\. Vertex Data Layout 최적화 (C++ 예시)



그래픽스 병목의 적지 않은 비율은 GPU 메모리 버스의 레이턴시에서 발생합니다. 정점 데이터 전송의 오버헤드를 줄이기 위해 구조체 멤버의 크기와 정렬 순서를 세심하게 조절해야 합니다.



AoS (Array of Structures) vs SoA (Structure of Arrays)



일반적인 정점 셰이더 연산 환경에서는 AoS 형태의 캐시 밀도가 우호적인 경우가 많으나, 위치 데이터만을 빠르게 필요로 하는 Shadow Pass 등의 상황에서는 위치 데이터만 별도의 스트림으로 빼내어 처리하는 구조가 최적화의 열쇠가 됩니다.



// 1. AoS 구조 (기본적인 렌더링 레이아웃으로 캐시 적중률에 좋음)

struct Vertex {

&#x20;   float x, y, z;    // Position (12 bytes)

&#x20;   float nx, ny, nz; // Normal (12 bytes)

&#x20;   float u, v;       // UV (8 bytes)

}; // Total: 32 bytes (Aligned to 4-byte boundaries)





3\. HLSL 셰이더에서의 보간 및 픽셀 연산



래스터라이제이션 단계에서 생성된 원근 교정 보간값(Perspective Correct Interpolation)을 바탕으로 프래그먼트 셰이더(Fragment Shader) 내부가 실행됩니다.



아래는 픽셀 셰이더 내부에서 단순 Phong Reflection 보간 방향을 제어하는 예제 코드입니다.



// Pixel Shader Input Struct

struct PS\_INPUT {

&#x20;   float4 position : SV\_POSITION;

&#x20;   float3 worldNormal : NORMAL;

&#x20;   float2 uv : TEXCOORD0;

};



// Simple Lambertian Diffuse Lighting 

float4 PSMain(PS\_INPUT input) : SV\_TARGET

{

&#x20;   float3 normal = normalize(input.worldNormal);

&#x20;   float3 lightDir = normalize(float3(0.5f, 1.0f, -0.3f));

&#x20;   

&#x20;   // Dot product for Lambertian reflection

&#x20;   float ndl = max(0.0f, dot(normal, lightDir));

&#x20;   float3 diffuseColor = float3(0.8f, 0.85f, 1.0f) \* ndl;

&#x20;   

&#x20;   return float4(diffuseColor, 1.0f);

}





4\. 파이프라인 최적화 체크리스트



Draw Call 최소화: Batching, GPU Instancing 기법을 사용하여 드로우콜 발행 시의 오버헤드 축소.



과도한 State 변경 지양: Render State(블렌딩, 컬링 등)의 빈번한 전환은 파이프라인 플러시(Flush)를 유발합니다.



Early-Z 검사 극대화: 픽셀 셰이더가 실행되기 전에 깊이 버퍼를 먼저 검사하여 가려질 픽셀 연산을 사전에 탈락시킵니다.
