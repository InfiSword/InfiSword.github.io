## 1. 구현 중심 포트폴리오 설계 원칙

팀 프로젝트(4인, 9인)가 포함되어 있으므로, 평가관은 **"이 많은 코드 중 지원자가 직접 짠 코드는 무엇인가?"**를 가장 먼저 확인합니다. 
따라서 단순히 시스템 소개나 게임의 룰을 나열하는 슬라이드는 모두 배제하고, 다음 **3장 구조**로 프로젝트를 압축합니다.

* **1장 (개요 및 구현 요약):** 프로젝트 정보 + 본인의 구체적인 역할(Role) + **내가 구현한 핵심 모듈 리스트(Core Components)**
* **2장 (내가 구현한 모듈 A 상세):** 직접 코딩한 모듈의 설계 의도, 클래스 다이어그램 및 핵심 C++/C# 코드 스니펫
* **3장 (내가 구현한 모듈 B 상세 및 피드백):** 또 다른 전담 구현 모듈의 알고리즘 흐름과 정량적 결과 지표, 문제점 극복 피드백

### 📌 Project 1: WinAPI Don't Starve 모작 (2 슬라이드)
> **구현 중심 키워드:** 밑바닥부터 구현한 2D 렌더러 최적화 및 그리드 공간 분할

* **03. [내가 구현한 것 ①] 2D 그리드 공간 분할 및 동적 갱신 시스템**
  - **내용:** 수천 개 객체 루프 부하를 막기 위해 전체 월드를 256px 셀 단위 정적 2D 배열(`m_spatialGrid`)로 나누고, 동적 객체 이동 시 경계를 넘어갈 때만 셀 위치를 변경하는 `UpdateObjectGrid()` 구현.
  - **시각화:** 클래스 헤더 구조 및 `UpdateObjectGrid` 핵심 코드 스니펫.
* **04. [내가 구현한 것 ②] 3x3 국소 쿼리 및 카메라 가시 범위 컬링**
  - **내용:** 카메라 뷰포트 영역의 3x3 인접 셀만 스캔하여 탐색 비용을 $O(N)$에서 $O(1)$로 단축하고, 다중 셀에 걸친 큰 오브젝트의 중복 연산을 1프레임 단위의 `m_spatialQueryStamp`로 원자적으로 걸러내는 쿼리 알고리즘(`QueryObjectsInRectArea`) 구현.
  - **시각화:** 3x3 그리드 가시화 다이어그램 + 10배 성능 향상 지표 수치 제시.

### 📌 Project 2: File Tower Defense (2 슬라이드)
> **구현 중심 키워드:** UI-World 좌표계 단일화 레이아웃 및 윈도우 스타일 이벤트 파이프라인 (4인 팀)

* **05. [내가 구현한 것 ①] 드래그&드랍 대응 GameObjectLayout 및 배치 매니저**
  - **내용:** 바탕화면 파일 배치 느낌을 주기 위해 기존 UI 기반(RectTransform) 설정을 제거하고, 월드 공간의 오브젝트 전용 레이아웃(`GameObjectLayout`) 및 동적 스크린 해상도 자동 맞춤(`FitToScreen`) 툴 직접 구현. 
  - **시각화:** `GridManager`의 원자적(Atomic) 배치 함수 `TrySetUnitAtGrid` C# 코드와 롤백 메커니즘.
* **06. [내가 구현한 것 ②] 중앙 집중식 입력 및 상호작용 이벤트 시스템**
  - **내용:** 수많은 파일 유닛에 개별 이벤트를 달지 않고, 중앙의 `InputManager`가 Raycast로 드래그/호버/클릭 상태를 감지해 인터페이스 `IInteractable`을 통해 한 번에 이벤트를 전파하는 `InteractionHandler` 구조 설계.
  - **시각화:** 마우스 입력 ➔ 핸들러 ➔ 유닛 이벤트 전파 흐름도.

### 📌 Project 3: Autonomous Racing Agent (3 슬라이드)
> **구현 중심 키워드:** Spline 기반 학습 환경 자동화 및 벡터 내적(Dot Product)을 활용한 방향 판정 (2인 팀)

* **07. [내가 구현한 것 ①] Spline 체크포인트 자동 생성 툴 및 학습 환경 설계**
  - **내용:** 맵 제작 수작업을 없애기 위해 Unity Spline 경로를 따라 구간을 자동 생성 및 분할하는 에디터 툴(`SplineToCheckpointGenerator`) 구현. 차량의 진행 상태(`CarProgress`)와 ML-Agents 연동.
  - **시각화:** 유니티 에디터 스크린샷 + 체크포인트 자동 할당 원리도.
* **08. [내가 구현한 것 ②] 벡터 내적(Dot Product) 기반 실시간 역주행 판정**
  - **내용:** 물리 충돌 후 방향을 잃어 역주행하는 에이전트를 위해 트랙 Spline의 진행 방향 벡터와 차량의 속도 벡터를 정규화 후 내적(Dot Product) 연산하여 실시간 정방향(+) 및 역방향(-) 상태를 판정하는 알고리즘 구현.
  - **시각화:** 벡터 내적 부호 판정 수학 공식과 방향 판정 알고리즘 코드.
* **09. [학습 지표 결과 및 Policy Loss 분석 피드백]**
  - **내용:** TensorBoard 학습 그래프(누적 보상 상승/엔트로피 감소) 분석 및 초기 엄격한 보상 스케일링으로 인해 발생한 Policy Loss 불안정성을 분석하고 하이퍼파라미터 튜닝(Batch Size 조정) 피드백 제공.

### 📌 Project 4: World First Kill (3 슬라이드)
> **구현 중심 키워드:** 리플렉션 기반 CSV 파서 및 PRNG 난수 흐름 복제 세이브 시스템 (9인 팀)

* **10. [내가 구현한 것 ①] 리플렉션 기반 CSV 파서 및 구글 시트 비동기 동기화**
  - **내용:** 런타임에 클래스 구조를 읽어 헤더와 맵핑하여 데이터 구조 추가 시 수동 코딩이 필요 없는 **리플렉션 기반 CSV 파서** 구현. UnityWebRequest를 활용해 구글 시트 데이터를 로컬 캐시와 대조하여 비동기 업데이트하는 버전 매니저 구축.
  - **시각화:** 리플렉션 파싱 코드 스니펫.
* **11. [내가 구현한 것 ②] PRNG(Splitmix64) Seed/Token 기반 세이브/로드 시스템**
  - **내용:** 방대한 맵/스킬/퀘스트 등의 데이터를 JSON 파일로 일일이 저장하지 않고, 64비트 대표 `BaseSeed`와 난수 소비 누적값(`Usage Count`)을 결합한 경량 텍스트 `Token`만으로 원래의 무작위성 상태를 100% 동일하게 재현하는 복구 모듈 설계.
  - **시각화:** `SaveLoadManager`의 토큰 직렬화 포맷 맵핑 다이어그램 및 세이브/로드 파이프라인.
* **12. [시드 비교 검증 및 String 해싱 개선 피드백]**
  - **내용:** 대표 시드 복원 검증 비교 스크린샷 + 기존 시드-토큰 방식의 한계점(초기화 순서 종속성)을 깨닫고, 퀘스트/맵 식별 문자열을 조합해 독립 시드를 추출하는 **String 해싱 기반 난수 전환 개선 로드맵** 제시.
