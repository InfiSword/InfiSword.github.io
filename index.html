1. UI 시스템에서 GameObject 기반으로의 아키텍처 전환

초기 유닛 설계를 RectTransform(UI) 기반으로 진행했으나, 적(Enemy) 유닛과의 상호작용 및 절대 좌표 추적에서 비효율성을 발견했습니다.

  • 문제점: 앵커(Anchor)와 피벗(Pivot)에 종속된 UI 좌표는 월드 공간의 물리 연산 시 잦은 좌표 변환(WorldToScreen 등)을 초래하여 성능 및 유지보수성을 저하시켰습니다.
  • 해결책: 모든 유닛을 GameObject 기반으로 전환하고 transform.position을 통해 절대 좌표를 관리하는 하이브리드 방식을 채택했습니다.
  • 성과: 유닛 배치, 타겟팅 로직, 물리 연산 시 불필요한 좌표 변환 연산을 제거하고 시스템의 직관성을 높였습니다.

2. 중앙 집중식 입력 시스템 (InputManager & InteractionHandler)

개별 유닛이 스스로 마우스 이벤트를 처리하던 구조를 탈피하여, 중앙 매니저가 입력을 통제하는 구조를 설계했습니다.

  • 상태 관리: None, Pressing, DraggingObject, DraggingBox 등 상태를 세분화하여 정교한 조작감을 구현했습니다.
  • UI 관통 클릭 방지: EventSystem.current.IsPointerOverGameObject() 및 레이캐스트 레이어 분리를 통해 UI 창 뒤의 유닛이 클릭되는 현상을 차단했습니다.
  • 추상화: IInteractable 인터페이스를 통해 파일, 폴더, 적 등 다양한 객체가 동일한 핸들러를 통해 일관된 방식으로 상호작용하도록 구현했습니다.

3. 그리드 기반 유닛 배치 시스템 (FileGridManager)

월드 좌표를 그리드 인덱스로 즉시 변환하고, 장애물 유무와 점유 상태를 판단하는 로직을 구축했습니다.

  • 동적 스냅: 유닛 드래그 종료 시 가장 가까운 빈 그리드(NotOccupied)를 탐색하여 자동으로 위치를 교정하는 로직을 구현했습니다.
