System Redesign Log

1. UI 시스템에서 GameObject 기반 아키텍처로의 전환

초기 설계에서 RectTransform(UI) 기반의 유닛 배치를 시도했으나, 월드 좌표 기반의 바이러스(적)와의 상호작용 및 렌더링 순서 관리에서 한계를 직면했습니다.

이를 해결하기 위해 모든 유닛의 베이스를 GameObject로 전환했습니다. 앵커와 피벗에 종속된 UI 좌표 대신 transform.position이라는 절대적 월드 좌표를 사용함으로써, 잦은 좌표 변환 연산을 제거하고 물리 연산의 정확도를 확보했습니다.

2. 중앙 집중식 입력 및 상호작용 (InteractionHandler)

수많은 파일 유닛이 개별적으로 마우스 이벤트를 수신하는 구조는 성능 저하와 입력 충돌을 야기했습니다. 이를 방지하기 위해 InputManager를 통한 입력 일원화를 구현했습니다.

  • 드래그 시작/진입/종료 상태를 매니저에서 통합 관리
  • 박스 드래그 다중 선택과 단일 클릭 이동의 상태 머신(FSM) 분리
  • UI 관통 방지 로직을 통해 윈도우 창 조작 시 유닛이 클릭되는 오류 차단

3. 그리드 기반 논리 시스템 (FileGridManager)

바탕화면의 아이콘 정렬 방식을 재현하기 위해 월드 좌표를 그리드 인덱스로 실시간 매핑하는 시스템을 구축했습니다.

  • 유닛 드래그 종료 시 가장 가까운 NotOccupied 그리드 자동 탐색 및 스냅
  • 동적 장애물(땅굴 등) 생성 시 해당 그리드의 점유 상태를 즉시 갱신하는 논리 구조 설계
