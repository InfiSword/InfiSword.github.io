PROJECT REPORT // FILE_TOWER_DEFENSE

File Tower Defense: 시스템 리팩토링 및 최적화

1. 아키텍처 전면 재설계 (UI → World)

기존 RectTransform(UI) 기반의 유닛 시스템을 GameObject(Transform) 기반으로 전환하여 좌표계 종속성을 제거하고 월드 좌표 연산 비용을 획기적으로 절감했습니다.

2. 공간 분할 최적화 (O(1) Search)

전체 그리드 탐색 대신 3x3 국소 영역 탐색을 도입하여 탐색 비용을 O(n)에서 O(1)로 단축, 약 10배 이상의 성능 향상을 달성했습니다.

3. 트랜잭션 기반 배치 시스템

원자적(Atomic) 배치 로직을 설계하여 유닛 이동/설치 시 데이터 불일치를 방지하고 실패 시 이전 상태로 안전하게 롤백되는 구조를 완성했습니다.

4. 중앙 집중식 입력 관리

InputManager와 InteractionHandler를 통해 수많은 유닛의 개별 이벤트를 제거하고 중앙에서 통합 관리하여 런타임 오버헤드를 최소화했습니다.

PROJECT REPORT // AUTONOMOUS_RACING_AGENT

Autonomous Racing Agent: 강화학습 및 주행 시스템

1. 다방향 레이캐스트 환경 감지

전방 9개, 후방 3개의 레이캐스트를 통해 실시간으로 도로와 벽을 인식하며, 정규화된 거리 데이터를 신경망의 관측 벡터(36차원)로 활용합니다.

2. 체크포인트 및 보상 설계 (Reward Engineering)

역주행 방지를 위한 순차적 체크포인트 시스템을 설계하고, 목표 거리 감소 보상과 안전성 페널티를 결합하여 안정적인 자율 주행을 구현했습니다.

3. 연속적 행동 제어 및 입력 스무딩

-1.0 ~ 1.0의 연속 행동 공간을 사용하며, 이전 프레임과의 변화량을 제한하는 스무딩 알고리즘을 적용하여 부드럽고 물리적인 주행을 실현했습니다.

4. 다중 트랙 기반 일반화 학습

스플라인 기반 체크포인트 자동 생성 시스템을 통해 다양한 트랙에서 랜덤하게 학습함으로써 단일 환경에 과적합되지 않는 일반화 성능을 확보했습니다.

PROJECT REPORT // WORLDFIRSTKILL

WorldFirstKill: 데이터 저장 및 파싱 시스템

1. 리플렉션 기반 CSV 파싱

런타임에 클래스 구조를 분석하여 CSV 헤더와 필드를 자동 매핑함으로써 수동 파싱 코드 없이 새로운 데이터 타입을 즉시 수용할 수 있는 확장성을 갖췄습니다.

2. 서버 기반 버전 관리 시스템

구글 스프레드시트 서버 버전과 로컬 캐시 버전을 비교하여 변경된 파일만 비동기(UnityWebRequest)로 업데이트하는 효율적인 데이터 관리 루틴을 구현했습니다.

3. Seed/Token 기반 상태 복원

단순 초기 시드 저장을 넘어, 영역별 난수 사용량(Usage Count)을 고정 폭 토큰으로 저장하여 세이브 로드 시에도 난수 흐름을 100% 동일하게 재현합니다.

4. JSON 데이터 직렬화 및 비동기 로딩

JsonUtility를 활용한 복합 데이터 구조의 저장/불러오기와 UI 블로킹 없는 비동기 로딩 시스템으로 부드러운 게임 경험을 제공합니다.

PROJECT REPORT // SLIME PROJECT

Slime Project: 상태머신 및 게임 로직 시스템

1. FiniteStateMachine 상태 관리

Enter/Exit/Update 라이프사이클을 갖는 정형화된 상태머신을 통해 모든 몬스터와 플레이어의 복잡한 행동 패턴을 체계적으로 설계했습니다.

2. 컴포지션 패턴 기반 플레이어 시스템

이동, 공격, 대시 등 각 기능을 독립적인 클래스로 분리하고 변신 형태에 따라 토끼(더블점프, 벽점프) 등 능력이 유연하게 교체되는 구조를 구축했습니다.

3. 델리게이트 기반 보상 AI 설계

보스 몬스터의 스킬을 델리게이트 배열로 관리하여 거리와 상황에 따라 지능적으로 스킬을 선택하고, HP 단계별 스킬 업그레이드 시스템을 적용했습니다.

4. 확장 가능한 몬스터 AI 아키텍처

공통 상태(Idle, Move, Attack)를 상속받아 몬스터별 특수 상태만 추가하면 되는 구조로, 개미, 곰, 거미 등 다양한 개체의 행동을 효율적으로 구현했습니다.

PROJECT REPORT // WINAPI_DONT_STARVE

WinAPI Don't Starve Clone: 시스템 최적화 및 렌더링

1. 공간 분할(Grid-based) 최적화

전체 월드를 그리드 단위로 분할 관리하여, 현재 화면에 보이는 영역(Viewfrustum)의 오브젝트만 연산하고 렌더링하는 효율적인 구조를 직접 구현했습니다.

2. WinAPI 렌더링 파이프라인 구축

GDI/GDI+를 활용한 더블 버퍼링 시스템을 구축하여 화면 깜빡임을 제거하고, 수많은 스프라이트 자원을 효율적으로 관리하는 리소스 매니저를 설계했습니다.

3. 컴포넌트 기반 객체 아키텍처

상속보다는 구성을 활용하여 이동, 충돌, 렌더링 등의 기능을 컴포넌트로 분리, 다양한 게임 오브젝트를 유연하게 생성하고 관리할 수 있도록 설계했습니다.

4. 상태 기반 AI 및 게임 루프

몬스터의 행동 패턴을 상태 머신으로 관리하고, 시간 흐름에 따른 환경 변화(낮/밤) 및 플레이어 생존 수치(허기, 체력) 시스템을 유기적으로 연결했습니다.
