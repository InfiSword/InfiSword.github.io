Architecture Redesign

1. UI(RectTransform)에서 GameObject(Transform) 기반으로의 전환

초기 단계에서는 윈도우 바탕화면의 느낌을 구현하고자 파일 유닛을 UI 시스템으로 구축했습니다. 하지만 프로젝트 규모가 커짐에 따라 다음과 같은 결정적인 설계적 한계를 마주했습니다.

  • 좌표계 종속성: RectTransform은 캔버스의 앵커(Anchor)와 피벗(Pivot) 설정에 따라 월드 좌표값이 상대적으로 변합니다. 이로 인해 유닛의 절대적인 위치를 파악하는 로직이 매우 복잡해졌습니다.
  • 연산 비용의 증가: 월드 좌표 기반의 바이러스(적)가 UI 기반의 파일 유닛 간의 거리 계산 및 물리적 충돌 판정 로직을 짜고자 할 때, 파일을 타겟팅하거나 파티클 이펙트를 출력할 때마다 Camera.WorldToScreenPoint, Camera.ScreenToWorldPoint 등의 변환 함수관련한 로직을 추가해 줘야 했습니다.
  • 계층 구조의 복잡성: 서로 다른 부모 객체(UI Hierarchy) 간의 위치 변환 로직은 코드의 가독성을 해치고 복잡한 로직의 구조가 나타났습니다.

이를 해결하기 위해 모든 유닛을 GameObject(Transform) 기반으로 전면 교체하여 월드 좌표계로 통일하여, 성능과 확장성, 로직을 간단하게 짜기 위한 이점들을 동시에 확보했습니다.

2. 그리드 시스템 아키텍처 (FileGridManager & FileGrid)

[FileGrid: 데이터 중심의 개별 셀 매니저]

유닛을 GameObject 기반으로 전환함에 따라, 윈도우 바탕화면의 아이콘 정렬 시스템을 모방하여 유닛 배치 시스템을 구축했습니다.

FileGrid는 타워(파일)를 설치할 수 있는 최소 단위인 'Slot' 역할을 수행하며, 해당 셀의 점유 상태와 인접 유닛 간의 시너지, 버프를 관리합니다.

FileGrid의 기본적인 구조는 아래와 같습니다.

[GameObjectGridLayout: GameObject 기반 커스텀 레이아웃]

화면 해상도에 맞춰 GameObject인 FileGrid의 간격과 셀 크기를 동적으로 계산하고, 전체 시스템의 시작 월드 좌표를 제공하는 핵심 클래스입니다.

public class GameObjectGridLayout : MonoBehaviour
{
    [Header("Grid Settings")]
    [SerializeField] private int columns = 13; // X축 그리드 수
    [SerializeField] private int rows = 7;     // Y축 그리드 수

    [SerializeField] private Vector2 cellSize = Vector2.one;
    [SerializeField] private Vector2 spacing = Vector2.zero;
    [SerializeField] private Vector2 padding = Vector2.zero;

    [Header("Layout Settings")]
    [SerializeField] private bool fitToScreen;

    public int Columns => columns;
    public int Rows => rows;
    public Vector2 CellSize => cellSize;
    public Vector2 Spacing => spacing;
    public bool GetFitToScreen => fitToScreen;
    public Vector2 StartPos { get; private set; }

   // ... 
    
    // 카메라 해상도에 맞춰 그리드 데이터 동적 계산
    public void FitToScreen() {
        Camera cam = Camera.main;
        float height = cam.orthographicSize * 2f;
        float width = height * cam.aspect;

        // 여백과 간격을 제외한 실제 가용 셀 크기 역산
        cellSize.x = (width - (padding.x * 2) - (spacing.x * (columns - 1))) / columns;
        cellSize.y = (height - (padding.y * 2) - (spacing.y * (rows - 1))) / rows;

        // 좌상단 기준 월드 시작 좌표 설정
        StartPos = new Vector2(-(width / 2) + padding.x + (cellSize.x / 2), 
                                (height / 2) - padding.y - (cellSize.y / 2));
    }
}

[FileGridManager: 전체 시스템 조율 및 물리-논리 변환]

FileGridManager는 모든 셀을 2D 배열(gridArray[x, y])로 관리하며, 게임 전체의 그리드 논리를 처리합니다.

  • 좌표 변환 시스템: WorldToGridIndex를 통해 마우스 위치를 즉시 인덱스로 변환하고, GetGrid를 통해 주변 3x3 영역 내 가장 가까운 그리드를 탐색하는 최적화 로직을 포함합니다.
  • 파일 유닛 배치 관리: TrySetUnitAtGrid 함수는 유효성 검사 후 기존 그리드에서 유닛을 제거, 새 그리드에 부모-자식 관계 설정 및 FileGrid.SetFileUnit() 호출을 통한 버프 자동 적용까지의 트랜잭션을 보장합니다.
  • 장애물 및 드롭 처리: 땅굴(WormHole)과 같은 동적 장애물 레이어를 관리하고, 폴더 객체 위로 드롭되는 특수한 상호작용(HandleDropAt)을 총괄합니다.
// FileGridManager.cs: 그리드 점유 상태 및 계층 구조 동기화
public bool TrySetUnitAtGrid(File_Base unit, FileGrid targetGrid) {
    if (targetGrid.obstacleObject != null) return false; // 장애물 체크
    
    // GameObject 기반 설계로 인한 직관적인 위치 및 계층 구조 동기화
    unit.transform.position = targetGrid.transform.position;
    unit.transform.SetParent(targetGrid.transform);
    targetGrid.SetFileUnit(unit); // 내부에서 버프 연산 자동 수행
    return true;
}

3. 중앙 집중식 입력 시스템 (InputManager)

수많은 유닛이 개별적으로 이벤트를 수신하지 않고, InputManager가 모든 마우스 입력을 통합 관리하여 IInteractable 인터페이스를 구현한 객체에 전달합니다.

// InputManager.cs: Raycast를 통한 중앙 집중식 상호작용 객체 탐지
private IInteractable GetInteractableUnderMouse() {
    // 월드 좌표상의 유닛과 바이러스를 레이어로 구분하여 정교하게 검출
    RaycastHit2D hit = Physics2D.Raycast(m_mouseWorldPosition, Vector2.zero, 0f, unitLayerMask);
    if (hit.collider != null) {
        return hit.collider.GetComponent<IInteractable>();
    }
    return null;
}

4. 동적 데이터 동기화 (FileGridManager)

유닛 배치 시 단순히 좌표를 맞추는 것이 아니라, 그리드 데이터와 유닛의 부모-자식 관계를 동기화하여 논리적 무결성을 유지합니다.

// FileGridManager.cs: 그리드 점유 상태 및 계층 구조 동기화
public bool TrySetUnitAtGrid(File_Base unit, FileGrid targetGrid) {
    if (targetGrid.obstacleObject != null) return false; // 장애물 체크
    
    // Transform 기반 설계로 인한 직관적인 위치 및 계층 구조 동기화
    unit.transform.position = targetGrid.transform.position;
    unit.transform.SetParent(targetGrid.transform);
    targetGrid.SetFileUnit(unit);
    return true;
}
