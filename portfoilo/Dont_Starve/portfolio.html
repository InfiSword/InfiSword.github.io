01

그리드 공간 분할 및 동적 이사 메커니즘

월드 맵 공간을 $256 \times 256$ 크기의 그리드 셀로 나눈 뒤, 각 오브젝트를 자신의 중심 좌표에 속하는 단 하나의 셀 포인터 배열(m_spatialGrid[GRID_WIDTH][GRID_HEIGHT])에 등록합니다.

💡 '자신'과 '중심 좌표'의 정의

여기서 자신이란 GameObject를 상속받아 월드에 존재하는 구체적인 캐릭터, 몬스터, 트리 등의 개체(Instance)를 가리킵니다. 중심 좌표는 개체의 AABB 바운딩 박스 중심점
centerX = X + Width * 0.5f, centerY = Y + Height * 0.5f를 뜻하며, 이 단일 점을 기준으로 소속 격자 셀을 식별합니다.

🔄 실시간 셀 업데이트 (이사/Migration) 과정
  1. 이동 이벤트 감지: 오브젝트의 위치/스케일 변경 시 Transform 컴포넌트가 SetSpatialDirty()를 트리거하여 m_isGridDirty = true(더티 플래그)를 설정합니다.
  2. 지연 동기화 (Lazy Evaluation): 프레임 도중 여러 번 좌표가 흔들리더라도 즉시 갱신하지 않고, LateUpdate() 시점에 단 1회만 최종 연산을 수행해 중복 연산을 완전 제거합니다.
  3. O(1) 조기 차단 (Early Exit): 새로운 중심점 좌표로 역산한 그리드 셀 인덱스(newX, newY)가 이전 셀 인덱스와 같다면, 리스트 조작 없이 즉시 연산을 종료합니다.
  4. 포인터 이사: 격자가 바뀐 경우에만 이전 셀 벡터에서 포인터를 삭제(std::remove)하고 새 셀 벡터에 push_back하여 이관을 완수합니다.
02

다단계 가시성 필터링 및 뷰포트 노출 흐름

카메라 가시 판정을 위해 단순 충돌 검사만을 사용하는 것이 아니라, 연산 효율에 맞춰 최적화된 다단계 필터링 구조를 취하고 있습니다.

1단계: 공간 분할 거친 범위 검색 카메라 뷰포트 사각형 영역과 겹쳐진 그리드 셀 목록 내의 객체들만 1차 후보군으로 압축 수집
2단계: AABB 교집합 검사 수집된 후보군 객체의 AABB(Axis-Aligned Bounding Box) 범위와 뷰포트 Rect의 교차 연산 수행
3단계: 실제 스프라이트 영역 가시 판정 스프라이트 스케일 및 피벗 기준의 세부 렌더 바운드로 최종 렌더링 판단 (IsObjectInViewport())
🏃‍♂️ 플레이어 이동 시 정적 사물(Static Object) 노출 흐름
  • 정적 사물 고정 등록: 나무, 바위 등은 게임 시작(또는 생성) 시점에 자신의 중심 좌표 격자 셀에 단 1회 등록되며, 이동하지 않으므로 추가적인 그리드 갱신 연산 비용이 전혀 들지 않습니다.
  • 카메라 뷰포트 이동: 플레이어가 월드를 이동함에 따라 카메라의 추적 로직(FollowTarget())이 작동하여 카메라의 월드 중심 좌표(m_cameraPos)가 갱신됩니다.
  • 쿼리 대상 격자 자동 시프트: 매 프레임 실시간 이동한 카메라 뷰포트 Rect(GetViewportWorldRect())를 그리드 격자 범위(startX ~ endX, startY ~ endY)로 변환합니다.
  • 가시화 및 컬링: 새롭게 쿼리 영역에 들어온 셀 내부의 정적 오브젝트들이 자동으로 탐색되어 화면에 렌더링되고, 화면에서 멀어진 셀의 정적 오브젝트들은 자동으로 탐색에서 배제(컬링)됩니다.
03

버퍼 재사용 최적화 (Memory Alloc Free)

매 프레임 마다 수십 마리의 몬스터나 뷰포트 객체를 쿼리할 때 발생할 수 있는 동적 힙 메모리 할당(malloc/free)의 높은 오버헤드를 막습니다.

주요 특징

CameraManager 및 ColliderManager의 멤버 변수로 m_queryBuffer 벡터를 선언하고, 게임 시작 시점에 reserve(2048)를 호출해 메모리를 미리 잡아놓습니다. 이후 쿼리 시에는 clear() 메서드를 사용해 기존 할당 주소를 보존하며 재할당 비용을 제로(0)에 가깝게 유지합니다.

04

타일 가시 영역 컬링 및 캐싱 (Tile Culling)

$100 \times 100$의 거대한 타일맵을 드로우할 때 뷰포트에 잡히는 활성 타일만 선별하여 드로우 콜을 발행하는 최적화입니다.

구현 메커니즘

  • 카메라의 뷰포트에 패딩 영역을 추가하여 렌더링 범위를 특정하고 타일 단위 인덱스를 바로 역산합니다.
  • 매 프레임 리소스의 재로드나 파괴를 최소화하기 위해 활성화 타일 비트맵을 해시맵인 m_tileCache로 구성하고, 뷰포트를 벗어난 항목만 한정적으로 지연 해제(CleanupUnusedTileCache)합니다.
ObjectManager.cpp - QueryObjectsInRect

🛠️ 시스템 아키텍처 개요

공간 분할(Spatial Partitioning) 기술과 뷰포트 컬링이 결합되어 실시간 렌더링 파이프라인의 병목을 어떻게 해소하는지 한눈에 보여주는 흐름도입니다.

1. 월드 맵 생성 전체 오브젝트를 좌표 기반으로 그리드 셀에 바인딩
➔
2. 카메라 업데이트 매 프레임 플레이어 기준의 가시 사각형(RectF) 정의
➔
3. 고속 셀 쿼리 해당 영역 내의 후보군 오브젝트만 그리드에서 즉시 획득
➔
4. 2차 필터링 및 렌더 피벗/스케일 영역 2차 필터링 후 렌더 커맨드 버퍼 전달

📈 렌더링 연산 비용 비교

브루트 포스 (기본 구현) - 연산량 최대 100% (매 프레임 전수 루프)
1차 공간 분할 쿼리 후 연산량 ~15% (인접 셀 후보군 선별)
2차 AABB & 3차 RenderBounds 필터링 통과 ~10% (실제 렌더 대상만 선별)

※ 월드 오브젝트가 약 1,500개 이상 배치되었을 시 60 FPS 유지를 위한 CPU 점유율이 약 70%에서 4% 미만으로 감소하는 극적인 프레임 유지력을 보였습니다.

💾 힙(Heap) 메모리 프레임당 할당 횟수

버퍼 매 프레임 지역 변수 생성 시 (비최적화) ~60회 / 프레임
버퍼 재사용 (reserve) 적용 시 (최적화) 0회 / 프레임 (사전 할당 완료)

지속적인 new, delete 또는 동적 컨테이너 확장을 억제하여, 가비지 컬렉션이 없는 C++ 환경에서의 메모리 파편화(Fragmentation)를 사전에 방지하여 롱런 타임 플레이 환경 안정성을 크게 개선하였습니다.
