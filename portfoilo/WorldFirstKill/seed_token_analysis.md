## 2. 시드 토큰(Seed Token) 물리 데코딩 테이블

전체 토큰 문자열은 총 **233자**로 구성되어 있으며, 규격화된 고정 자릿수 파싱 규칙에 따라 다음과 같이 14개의 독립 RNG 도메인 정보로 명확하게 분해됩니다.

### 1️⃣ 공통 헤더 영역 (23자)
* **Base Seed (앞 20자)**: `00000000000558290964` ➔ **`558,290,964`** (기본 결정적 시드값)
* **Domain Count (다음 3자)**: `014` ➔ **`14`** (활성화되어 있는 서브 RNG 도메인 개수)

## 3. 세이브 상태 복원 프로세스 전개 (How it is Parsed)

[SaveLoadManager.cs](file:///C:/Ptf/InfiSword.github.io/codeReference/WorldFirstKill/SaveLoadManager.cs)의 `RestoreUsageCount`가 본 JSON 데이터를 로드하면 메모리 상에서 다음과 같이 복원 프로세스가 전개됩니다.

```
[복원 파이프라인]

 1. 토큰 데이터 파싱
    - BaseSeed = 558290964 추출
    - 도메인 개수 = 14개 감지

 2. 루프를 돌며 각 도메인 초기화 및 Fast-Forward
    For 각각의 도메인 (1 to 14):
      - ex) 도메인 1 (Map, ID=1, Count=25):
        a. 서브 시드 계산: DeriveSubSeed(558290964, 1) = 931,048,252 (임의의 혼합값)
        b. RNG 생성: System.Random mapRng = new System.Random(931048252);
        c. Fast-Forward 루프:
           for (ulong i = 0; i < 25; i++) {
               mapRng.Next(); // 25번 난수를 흘려보내 스트림 위치를 정확히 일치시킴
           }
        d. GameSeed.domainRng[Domain.Map]에 등록 완료.

 3. 게임 재개
    - 플레이어가 새로운 맵 구역 진입 시 GameSeed.NextInt(Domain.Map, ...) 호출
    - 26번째 난수 스트림 값이 즉시 반환되어 세이브 시점 직후의 결정적 월드 생성 보장.
```

## 4. 아키텍처적 평가 및 가치
1. **완벽한 격리성 (No Crosstalk)**:
   - 직업군 조합(`Class`)과 스킬 분화(`Skill`) 등 8개의 전투 도메인이 개별 ID(100~103, 200~203)로 완벽히 분리되어 있어, 인게임에서 특정 직업의 스킬을 몇 번 셔플하든 맵 생성(`Map`)이나 보상(`Reward`)의 다음 난수 스트림이 밀려버리는 오염 현상이 전혀 일어나지 않습니다.
2. **세이브 파일 용량 극대화**:
   - 215개의 퀘스트 클리어 현황, 431개의 아이템 드롭 풀, 14개 직업 조합 상태 등 실제 객체 정보로 직렬화했다면 최소 **수십 KB에서 수 MB**가 될 정보를 단 **233자(약 0.2KB)**의 토큰으로 압축 보존함으로써 클라우드 세이브 패킷 및 디스크 입출력을 혁신적으로 절감했습니다.
