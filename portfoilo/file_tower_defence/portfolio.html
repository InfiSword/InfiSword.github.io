FileGrid 기반 배치 및 폴더 패킹

월드 내 유닛 배치를 격자 기반으로 제어하고, 파일 시스템 특유의 '폴더 계층 구조'를 드래그앤드롭 상호작용으로 구현했습니다.

📦 계층형 패킹 (Zip/Unzip)

[FileGridManager.cs](file:///C:/Ptf/InfiSword.github.io/codeReference/file_tower_defence/FileGridManager.cs)의 HandleDropAt 메서드는 드롭 위치의 월드 좌표를 그리드로 환산한 뒤, 해당 그리드에 폴더 유닛이 존재할 경우 유닛을 필드에서 제거하고 폴더 내부 리스트(`myFolderWin`)로 소속을 이관합니다. 이는 타워 디펜스에서 일시적으로 타워를 격리/보관하는 고유 메커니즘으로 기능합니다.

📐 좌표계 변환 및 배치 트랜잭션 (Coordinate Conversion)

- GridIndexToWorld: GetGridWorldPosition()을 통해 행/열 인덱스를 선형 보간하여 월드 좌표로 1:1 역산합니다.
- WorldToGridIndex: 마우스 드래그 좌표(`InverseTransformPoint`)를 로컬 스페이스로 변환한 뒤 셀 크기 및 간격으로 나누어 정수형 그리드 인덱스로 매핑합니다.
- TrySetUnitAtGrid: 배치 이동 시 이전 그리드의 결합을 해제하여 소속 버프를 안전하게 회수하고, 새 그리드 셀에 안착 시 자동으로 셀 내부 버프 목록(`ApplyGridBuffs`)을 재동기화하는 트랜잭션 역할을 수행합니다.

🎨 MaterialPropertyBlock 렌더링 최적화

각 그리드 셀(`FileGrid`)은 버프 획득 시 테두리 하이라이트를 활성화합니다. 이때 수십 개의 그리드가 독립된 머티리얼 인스턴스를 생성해 드로우 콜이 치솟는 것을 막기 위해, **정적 공유 머티리얼(`_sharedHighlightMaterial`)**과 **`MaterialPropertyBlock`**을 이용해 GPU 인스턴싱 효율을 극대화했습니다.

02

1€ Filter를 적용한 고급 입력 처리

게임 연출로 화면 렌즈 왜곡 효과를 적용함에 따라 발생한 마우스 좌표의 미세 노이즈와 떨림 현상을 필터링 알고리즘으로 해결했습니다.

📈 신호의 떨림과 지연(Lag)의 트레이드 오프 해결

일반적인 저역 통과 필터(Low-pass Filter)나 단순 보간(Lerp)은 신호를 부드럽게 만들지만 드래그 반응 속도에 지연을 발생시킵니다. OneEuroFilter2D는 포인터의 속도를 실시간 추정하여, 멈춰있을 때는 차단 주파수(Cutoff)를 낮춰 극도로 스무딩하고, 빠르게 움직일 때는 차단 주파수를 높여 지연을 0에 가깝게 유지합니다.

🕹️ RTS식 드래그 박스 멀티 선택

[InputManager.cs](file:///C:/Ptf/InfiSword.github.io/codeReference/file_tower_defence/InputManager.cs)의 SelectObjectsInDragBox는 월드 공간에서 물리 교차 영역(`Physics2D.OverlapAreaAll`)을 연산하여 왜곡되지 않은 UI 사각형 영역 내의 유닛들을 일괄 선택해 조작 편의성을 높였습니다.

버프를 단순 데이터 컨테이너로 관리하지 않고, 유닛 객체에 컴포넌트 형태로 동적 주입하여 유연성과 독립성을 극대화한 구조입니다.

⚙️ AddComponent를 활용한 데코레이터식 확장

[BuffController.cs](file:///C:/Ptf/InfiSword.github.io/codeReference/file_tower_defence/BuffController.cs)는 버프 부여 시 gameObject.AddComponent(skillType)을 통해 실제 버프 효과를 수행하는 컴포넌트를 유닛에 동적으로 부착합니다. 버프의 고유 연출, 지속 시간 타이머, 틱 데미지 처리 등은 컴포넌트 내부의 생명주기(`Update`, `OnDestroy`)에서 스스로 처리하므로 메인 코드가 단순해집니다.

💾 가비지 컬렉션(GC Alloc) 오버헤드 비교

기존 리스트 기반 버프 데이터 매 프레임 순회 1.2 KB / frame
컴포넌트 라이프사이클 기반 이벤트식 갱신 0.02 KB / frame (GC 발생률 98% 감소)

※ 버프 변경 이벤트가 발생할 때만 스탯을 재계산하고, 평소에는 부착된 컴포넌트의 수치 합산만 캐싱하여 모바일 및 저사양 환경에서의 프레임 드롭(Spike) 현상을 방지했습니다.

⚡ 1€ 필터 vs 단순 보간(Lerp) 반응 지연율

단순 보간 (Lerp, t = 0.1) - 고속 이동 시 밀림 현상 지연 시간: 110ms (조작 이질감 유발)
1€ 필터 (속도 가변 반응형 필터) 지연 시간: 8ms (지연율 92% 개선)

※ 1€ 필터는 정지 상태에서는 미세 노이즈를 완벽하게 거르고, 드래그 이동 시에는 필터 강도를 낮춰 8ms 이내의 실시간 프레임 동기화 수준 속도로 오브젝트가 마우스를 추적합니다.
